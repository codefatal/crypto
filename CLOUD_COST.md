# AutoCrypto 클라우드 운영 비용 추정

> 기준: 업비트 KRW 마켓, 15분봉, 상위 30 심볼, 24시간 연속 운영
> 환율: 1 USD = 1,400 KRW (참고용)

---

## 1. AWS EC2 (서울 리전 ap-northeast-2)

### 추천 인스턴스

| 용도 | 인스턴스 | vCPU | RAM | 시간당 | 월간 (On-Demand) |
|---|---|---|---|---|---|
| 최소 운영 | t3.micro | 2 | 1 GB | $0.0116 | **~$8.5** |
| 권장 (여유 있음) | t3.small | 2 | 2 GB | $0.023 | **~$16.8** |
| 고부하 (바이낸스 전종목) | t3.medium | 2 | 4 GB | $0.046 | **~$33.6** |

> asyncio 단일 프로세스 봇 → t3.small로 충분. 바이낸스 멀티스트림(100+ 심볼)이면 t3.medium 권장.

**Savings Plans / Reserved Instance 적용 시 약 40~60% 절감 가능** (1년 약정 기준)

### 데이터 전송 비용 (Outbound)

AWS는 인터넷 아웃바운드 트래픽에 과금합니다. 인바운드(수신)는 무료.

#### 트래픽 추정 (업비트 기준)

| 항목 | 빈도 | 패킷 크기 | 일일 |
|---|---|---|---|
| WebSocket ticker 수신 | ~200 msg/초 × 86,400초 | ~0.3 KB/msg | **~5 GB 수신** (무료) |
| REST OHLCV 갱신 | 30 심볼 × 96회/일 | ~5 KB/응답 | **~14 MB 수신** (무료) |
| Groq API 호출 | 최대 ~28회/일 | ~10 KB/왕복 | **~0.3 MB** |
| Discord/Telegram 알림 | ~50회/일 | ~2 KB/건 | **~0.1 MB** |
| **아웃바운드 합계** | | | **≈ 0.5 MB/일** |

**아웃바운드 0.5 MB/일 × 30일 = ~15 MB/월**
→ AWS 무료 티어(월 100 GB 아웃바운드) 내 → **$0**
→ 무료 티어 초과 시: $0.126/GB → **$0.002/월 (사실상 무시)**

#### 실제 네트워크 비용

| 항목 | 월 비용 |
|---|---|
| 데이터 전송 | **$0** (무료 티어 내) |
| EC2 t3.small | **~$16.8** |
| **합계** | **~$16.8 / 월 (~23,500 KRW)** |

## 2. NCP (Naver Cloud Platform, 한국 리전)

### 추천 서버

| 용도 | 서버 타입 | vCPU | RAM | 월간 (Standard) |
|---|---|---|---|---|
| 최소 운영 | c2-g3-s50 | 2 | 4 GB | **~15,000 KRW** |
| 권장 | c2-g3-s100 | 2 | 8 GB | **~22,000 KRW** |

> NCP는 국내 리전 → 업비트 API 레이턴시 유리 (RTT ~5ms vs AWS 서울 ~10ms)

### 데이터 전송 비용

NCP 기본 제공 트래픽: **월 1 TB 무료 아웃바운드** (컴퓨팅 서버 포함 시)

봇 실제 아웃바운드 ~15 MB/월 → **무료 범위 내, $0**

### 실제 네트워크 비용

| 항목 | 월 비용 |
|---|---|
| 데이터 전송 | **₩0** (기본 제공 내) |
| 서버 (c2-g3-s50) | **~₩15,000** |
| **합계** | **~₩15,000 / 월** |

## 3. AI API (Groq) 비용

| 티어 | TPD 한도 | 비용 |
|---|---|---|
| Free (on_demand) | 100,000 토큰/일 | **$0** |
| Groq API 유료 (llama-3.3-70b) | 무제한 | $0.59/1M input + $0.79/1M output |

**현재 설정 기준 (MAX_SYMBOLS_PER_CANDLE=30, AI_MIN_SCORE=70):**
- 예상 일 호출 수: ~5~15회 (강한 신호가 있는 심볼만)
- 예상 일 토큰 사용: ~5,000~30,000 tokens
- **무료 티어(100,000 TPD) 내 처리 가능 → $0**

## 5. 바이낸스 전종목 모드 시 추가 고려사항

바이낸스 `ACTIVE_EXCHANGE=binance`로 전환하면 트래픽이 크게 증가합니다.

| 항목 | 업비트 | 바이낸스 (전종목) |
|---|---|---|
| WebSocket 스트림 수 | 1개 (ticker 멀티) | ~20개 (25 심볼씩 멀티스트림) |
| 초기 히스토리 요청 | ~30개 | ~500개 (MAX_SYMBOLS 기본값) |
| 일일 수신 트래픽 | ~5 GB | ~20~50 GB |
| 권장 인스턴스 | t3.small | t3.medium |

> **바이낸스 전종목 사용 시 `MAX_SYMBOLS_PER_CANDLE=50~100`으로 제한 권장**
> (기본 500개 전종목은 Groq TPD 예산을 1일 만에 소진)

## 6. 비용 최적화 팁

1. **AWS Spot Instance** 사용 시 On-Demand 대비 70~90% 절감 가능
   - 단, 봇 중단 시 재시작 로직 필요 (`--interruption-behavior=stop` 권장)
2. **NCP 예약 서버** 1년 약정 시 약 30% 할인
3. **SQLite → PostgreSQL 전환 불필요** — SQLite로도 1년치 데이터 수백 MB 이내
4. **Groq 무료 티어 초과 시**: `MAX_SYMBOLS_PER_CANDLE`을 낮추거나 `AI_MIN_SCORE`를 높여 호출 빈도 감소
